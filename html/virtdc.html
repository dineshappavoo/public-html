Introduction.

Virtdc is an API for virtual machine placement and scaling which provides an environment to create, manage and monitor virtual machines effectively. Virtdc framework provides API to create virtual machines, maintain information about guest and host, terminate virtual machine and to handle static virtual machine placement. Internally it uses libvirt management API to accomplish this.

The following are the core functionalities of the virtdc api

create creates new domain from the base image
$virtdc create -h
usage: virtdc create [-h] vmid cpu memory maxmemory io

positional arguments:
  vmid        get the vmid
  cpu         get the cpu
  memory      get memory in KiB
  maxmemory   get maximum memory in KiB
  io          get the io in KiB
terminate terminates the running domain
$virtdc terminate -h
usage: virtdc terminate [-h] vmid

positional arguments:
  vmid        get the vmid
list lists running domain in all hosts
$virtdc list -h
usage: virtdc list [-h]
dominfo gives the domain information
$virtdc dominfo -h
usage: virtdc dominfo [-h] vmid

positional arguments:
  vmid        get the domain id
hostinfo gives the host information
$virtdc hostinfo -h
usage: virtdc hostinfo [-h] hostname

positional arguments:
  hostname    get the host
force-migrate migrates domain from source host to dest host
$virtdc force-migrate -h
usage: virtdc force-migrate [-h] vmid sourcehost desthost

positional arguments:
  vmid        get the domain
  sourcehost  get the source host
  desthost    get the dest host
removehost removes the host from the cluster
$virtdc removehost -h
usage: virtdc removehost [-h] hostname

positional arguments:
  hostname    get the host
loadbalance loadbalances all the hosts in the cluster
$virtdc loadbalance -h
usage: virtdc loadbalance [-h] hostname

positional arguments:
  hostname    get the host
consolidate consolidates domains with the minimal number of hosts
$virtdc consolidate  -h
usage: virtdc consolidate [-h]
addhost adds new host to the cluster
$virtdc addhost  -h
usage: virtdc addhost [-h] hostname cpu memory io

positional arguments:
  hostname    get the host
  cpu         get the cpu
  memory      get memory in KiB
  io          get the io in KiB
getip gets domain ip
$virtdc getip  -h
usage: virtdc getip [-h] vmid

positional arguments:
  vmid        get the domain id
setsmtpserver sets smtp server to send emails
virtdc setsmtpserver  -h
usage: virtdc setsmtpserver [-h] serverip

positional arguments:
  serverip    get the smtp server ip
setfrommailaddress sets from mailaddress to send emails
$virtdc setfrommailaddress  -h
usage: virtdc setfrommailaddress [-h] mailid

positional arguments:
  mailid      get the from mail address
addsupportmail adds support mail address to send mails in case of any exception
$virtdc addsupportmail  -h
usage: virtdc addsupportmail [-h] mailid

positional arguments:
  mailid      get the mail address
monitorcpu monitors cpu usage
$virtdc monitorcpu  -h
usage: virtdc monitorcpu [-h] vmid

positional arguments:
  vmid        get the domain id
monitormemory monitors memory usage
$virtdc monitormemory  -h
usage: virtdc monitormemory [-h] vmid

positional arguments:
  vmid        get the domain id
monitorio monitors io usage
$virtdc monitorio  -h
usage: virtdc monitorio [-h] vmid

positional arguments:
  vmid        get the domain id

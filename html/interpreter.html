A Simple Interpreter Developed in Prolog

By Dinesh Appavoo, December 10, 2014

Want To Write An Interpreter?

In this article you will know about how to create a basic interpreter for a language. Here are the three important components required to write an interpreter

  • Definite Clause Grammar
  • Parse Tree
  • Denotational Semantics
The following grammar [Right Recursive] is used to develop an interpreter. This grammar parses
  • variable declaration and initilization
  • full if condition
  • while loop
Based on the grammar the parse tree has to be generated. It is an ordered, rooted tree that represents the syntactic structure of a string according to some context-free grammar.
Parse trees are usually constructed according to one of two competing relations, either in terms of the constituency relation of constituency grammars (= phrase structure grammars) or in terms of the dependency relation of dependency grammars.

Prolog code block to generate the parse tree

Denotational semantics is an approach of formalizing the meanings of programming languages by constructing mathematical objects (called denotations) that describe the meanings of expressions from the languages. Mathmatical objects are constructed from the parse tree which will be given to the semantic producer as an input.

Code block for the denotational semantics

To test the interpreter, you can simply run the main method in the denotaional semantics code block. It will internally generate the parse tree for the input code and do the evaluation. This interpreter will allow the program to take two inputs x and y and do the operations on x and y which can invlove z, u and v.
